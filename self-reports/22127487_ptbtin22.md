## 6. Tự đánh giá & rút kinh nghiệm (tùy chọn)

### Điều bạn làm tốt:

- Chủ động nghiên cứu và tích hợp thành công luồng Omnichannel (đặc biệt là Zalo API) và Customer Management.
- Kịp thời phát hiện và fix dứt điểm các lỗi liên quan đến code generation (build_runner, JSON serialization) để không block CI/CD của nhóm.

### Khó khăn bạn gặp phải:

- Việc đồng bộ state liên tục khi tích hợp các nền tảng chat (như Zalo) đôi khi gây khó khăn do API có nhiều case cần handle.
- Lỗi out-of-sync của các file `.g.dart` (do build_runner sinh ra) và các conflict code lúc merge branch nhiều lần làm mất thời gian debug.

### Bạn sẽ làm khác đi điều gì nếu được làm lại:

- Sẽ thiết lập các rule chặt chẽ hơn cho linter và setup pre-commit hooks (chạy `dart fix` và `flutter test`) ngay từ ngày đầu dự án để không bị tốn công fix về sau.
