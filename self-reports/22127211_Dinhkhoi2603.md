## 6. Tự đánh giá & rút kinh nghiệm (tùy chọn)

> Phần optional, không tính điểm — nhưng giúp thầy hiểu hơn về quá trình bạn làm việc.

### Điều bạn làm tốt:

> - Hoàn thành đầy đủ 3 ticket được giao (Prompt Management UI + Mock, Google Analytics, Prompt Management API Integration) với code có cấu trúc Clean Architecture rõ ràng: domain → data → presentation.
> - Áp dụng nhất quán pattern MobX cho state management, `Either<Failure, T>` cho error handling, và `getIt` cho DI.
> - Khi tích hợp API thật ở Phase 2, vẫn giữ được mock fallback giúp UI hoạt động ngay cả khi backend chưa sẵn.
> - Tích hợp Firebase Analytics với event taxonomy đầy đủ (screen view, user_login, ticket_created, message_sent...) + user properties cho segmentation.

### Khó khăn bạn gặp phải:

> - CI bị gãy do mismatch version Flutter 3.29 vs SDK mới hơn (vấn đề `intl` package, const expression). Phải debug + bump dependency để fix.
> - Merge conflict liên tục khi rebase branch tích hợp Prompt vào main (nhiều người cùng động đến `repository_module.dart`, `store_module.dart`).
> - File `main.dart`, `routes.dart` trên main branch đã bị conflict tồn đọng từ nhiều branch khác — phải dọn dẹp lại duplicate imports / dead code khi làm task analytics.

### Bạn sẽ làm khác đi điều gì nếu được làm lại:

> - Pin Flutter SDK version chính xác cho dev + CI ngay từ đầu, không dùng range rộng → tránh CI break.
> - Tách commit nhỏ hơn, theo nguyên tắc 1 commit = 1 logical change để dễ review/rollback.
> - Viết test (unit + widget test) cho store và repository ngay khi implement, không để dồn về cuối.
