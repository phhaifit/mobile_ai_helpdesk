## 2. Architecture Design

### 2.1 Clean Architecture Layers

```
┌─────────────────────────────────────────────────────┐
│           PRESENTATION LAYER                        │
│  Screens, Widgets, MobX Store (State Management)   │
├─────────────────────────────────────────────────────┤
│           DOMAIN LAYER                              │
│  Entity (Customer, SupportTicket)                   │
│  Repository Interface (Abstract)                    │
├─────────────────────────────────────────────────────┤
│           DATA LAYER                                │
│  Repository Implementation                          │
│  DataSource (Mock In-Memory Database)               │
├─────────────────────────────────────────────────────┤
│           DI LAYER                                  │
│  Service Locator (get_it)                           │
└─────────────────────────────────────────────────────┘
```

### 2.2 State Management Flow

```
User Action (UI)
    ↓
MobX Action (CustomerStore)
    ↓
Repository Method Call
    ↓
DataSource CRUD Operation (In-Memory)
    ↓
ObservableList Updated
    ↓
Widget Rebuilds (Flutter MobX)
```

## 6. Offline Flow Design

### 6.1 Architecture: Offline-First

```
┌─────────────────────────┐
│   UI (Widgets)          │
├─────────────────────────┤
│   MobX Store            │
│   (Reactive State)      │
├─────────────────────────┤
│   Repository            │
│   (Business Logic)      │
├─────────────────────────┤
│   DataSource            │
│   (In-Memory List)      │ ← ALL OPERATIONS HERE (Offline)
│   - customers: List[]   │
│   - tags: List[]        │
└─────────────────────────┘
```

### 6.2 CRUD Operations (All In-Memory)

#### **Create Customer**
```
Flow: CustomerAddEditScreen → Store.addCustomer() → DataSource.addCustomer()
- Generate unique ID (timestamp-based)
- Add to _customers list
- Return created customer
- UI updates via observable list
```

#### **Read Customer List**
```
Flow: CustomerListScreen → Store.filteredCustomers (computed)
- Get from DataSource.getCustomers()
- Apply search filter (fullName, email, phone)
- Apply tag filter (selectedTagFilters)
- Exclude blocked (on Active tab)
- Return filtered list
```

#### **Update Customer**
```
Flow: CustomerDetailScreen/AddEditScreen → Store.updateCustomer() → DataSource.updateCustomer()
- Find customer by ID
- Replace in _customers list
- Notify observers
- Update UI
```

#### **Delete Contact Info**
```
Flow: CustomerDetailScreen → Remove email/phone → Update customer
- Clear specific field
- Update in store
- No hard delete (soft update)
```

#### **Merge Customers**
```
Flow: CustomerMergeScreen → Store.mergeCustomers(primaryId, secondaryId)
1. Get both customers
2. Create merged customer:
   - name from primary
   - email from primary OR secondary
   - phone from primary OR secondary
   - tags = union of both
3. Remove secondary from list
4. Replace primary with merged
5. Remove secondary from list
```

#### **Block/Unblock Customer**
```
Flow: CustomerListScreen → Store.blockCustomer() / unblockCustomer()
- Update isBlocked flag
- Move between tabs
```

#### **Search**
```
Flow: CustomerListScreen (search input) → Store.setSearchQuery()
- Computed filteredCustomers property filters by:
  - customer.fullName.contains(query)
  - customer.email.contains(query)
  - customer.phoneNumber.contains(query)
- Real-time as user types
```

#### **Filter by Tags**
```
Flow: CustomerFilterSheet → Store.toggleTagFilter()
- Maintain selectedTagFilters observable list
- Computed filteredCustomers only includes customers with ANY selected tag
- "Áp dụng" applies filters
- "Xóa tất cả" clears filters
```

## 7. Key Features Implementation

### 7.1 Email Validation

**Location:** `CustomerAddEditScreen._validateEmail()` and `CustomerDetailScreen._validateEmail()`

```dart
String? _validateEmail(String value) {
  if (value.isEmpty) return null;
  final re = RegExp(r'^[\w\-.+]+@[a-zA-Z\d\-.]+\.[a-zA-Z]{2,}$');
  return re.hasMatch(value) ? null : 'Email không hợp lệ';
}
```

**Inline Validation:**
- Shows ✓ (green) icon if valid
- Shows ✗ (red) icon if invalid
- Real-time as user types
- Prevents save if invalid

### 7.2 Search Implementation

**Location:** `CustomerStore.filteredCustomers` (computed)

```dart
@computed
List<Customer> get filteredCustomers {
  return customers.where((c) {
    if (c.isBlocked) return false;  // Exclude blocked on main list
    
    final matchesSearch = searchQuery.isEmpty || 
      c.fullName.toLowerCase().contains(searchQuery.toLowerCase()) ||
      (c.email?.toLowerCase().contains(searchQuery.toLowerCase()) ?? false) ||
      (c.phoneNumber?.contains(searchQuery) ?? false);
    
    final matchesFilter = selectedTagFilters.isEmpty ||
      c.tags.any((tag) => selectedTagFilters.contains(tag));
    
    return matchesSearch && matchesFilter;
  }).toList();
}
```

### 7.3 Filter Implementation

**Location:** `CustomerStore` with MobX actions

```dart
ObservableList<String> selectedTagFilters = ObservableList();

@action
void toggleTagFilter(String tag) {
  if (selectedTagFilters.contains(tag)) {
    selectedTagFilters.remove(tag);
  } else {
    selectedTagFilters.add(tag);
  }
}

@action
void clearTagFilters() {
  selectedTagFilters.clear();
}
```

### 7.4 Tag Management

**Features:**
- View existing tags as chips (with delete icon)
- Add new tags via input + button
- Assign multiple tags to one customer
- System-wide tag list (`allAvailableTags` computed property)
- Custom tags creatable on-the-fly

### 7.5 Segmentation

**Available Segments:**
- VIP

- Mới
- Tiềm năng

**Usage:**
- Set in `CustomerAddEditScreen` via DropdownButton
